# 🚀 Solidity Beginner Challenges

Welcome! This repository is designed to help you practice the fundamentals of Solidity step by step.

If you're new to blockchain or smart contracts — you're in the right place 💪🏽

---

## 🧠 What You'll Learn

Through these challenges, you will practice:

- Writing smart contracts in Solidity  
- Understanding variables and data types  
- Writing and using functions  
- Using mappings (storing data per user)  
- Working with structs (complex data)  
- Implementing basic access control  

---

## 🛠️ Where to Write Your Code

We will be using Remix IDE:

👉 https://remix.ethereum.org

No installation needed — it runs in your browser.

---

## 📌 How to Use This Repository

Each challenge contains:

- A starter smart contract  
- Instructions (README inside each folder)  
- A task for you to complete  

👉 Your job is to:
- Read the instructions  
- Complete the missing function(s)  
- Test your code in Remix  

---

## 🔥 Challenges

### 🧩 Challenge 1 — Data Types & Variables  
Learn how to store and update basic data types like `uint`, `string`, and `bool`.

---

### 🧩 Challenge 2 — Functions  
Understand how to write functions, pass inputs, and return values.

---

### 🧩 Challenge 3 — Mappings  
Store and manage data for multiple users using `mapping` and `msg.sender`.

---

### 🧩 Challenge 4 — Structs  
Work with structured data by grouping related variables and storing them per user.

---

### 🧩 Challenge 5 — Access Control  
Learn how to restrict function access using `require` and `msg.sender`.

---

## 🎯 Goal

By the end of these challenges, you should be able to:

- Write and deploy a simple smart contract  
- Understand how data is stored on-chain  
- Handle multiple users in your contract  
- Apply basic security practices  
- Build the foundation needed for hackathon projects 🚀  

---

## 💡 Tips for Success

- Don’t just copy code — experiment and try your own variations  
- Break things and fix them — that’s how you learn  
- Take your time to understand each concept before moving on
