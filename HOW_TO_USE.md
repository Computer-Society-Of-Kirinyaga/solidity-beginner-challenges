# 🧭 How to Write and Test Your Smart Contracts (Beginner Guide)

This guide will walk you step by step on how to use Remix IDE and connect it with this repository.

---

## 🟣 Step 1: Open Remix IDE

Go to:

👉 https://remix.ethereum.org

---

## 🟣 Step 2: Create a New File

1. On the left sidebar, click **File Explorer**
2. Click **New File**
3. Name it something like:
   DataTypesChallenge.sol

---

## 🟣 Step 3: Copy Starter Code

1. Go to this GitHub repository  
2. Open the challenge file  
3. Copy the code  
4. Paste it into Remix  

---

## 🟣 Step 4: Complete the Challenge

Follow the instructions in the challenge README.

👉 Example:
- Write a function  
- Update variables  
- Use mappings or structs  

---

## 🟣 Step 5: Compile Your Code

1. Click **Solidity Compiler** (left panel)
2. Click **Compile**

👉 Fix any errors if they appear

---

## 🟣 Step 6: Deploy Your Contract

1. Click **Deploy & Run Transactions**
2. Select:
   - Environment: **Remix VM**
3. Click **Deploy**

---

## 🟣 Step 7: Interact with Your Contract

After deploying, you will see your contract:

- Call functions  
- Input values  
- Read stored data  

---

## 🟣 Step 8 (Optional): Push Your Code to GitHub

If you want to save your solution:

1. Copy your final code  
2. Go to your GitHub account  
3. Create a new repository  
4. Paste your code and commit  

---

## 🧠 Key Concepts to Remember

- `msg.sender` → the user interacting with the contract  
- `mapping` → stores data per user  
- `struct` → groups related data  
- `public` → makes variables accessible  

---

## 🚀 Final Advice

Don’t rush.

Try things. Break things. Fix things.

That’s how developers are made.
